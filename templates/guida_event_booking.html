🎯 1. Introduzione

EventBooking e' un'applicazione web per la gestione e prenotazione di posti per eventi in sale conferenza, teatri, cinema o spazi simili.

Caratteristiche principali

  • Calendario interattivo con navigazione per anno e mese
  • Gestione multi-sale con layout personalizzabili (file x colonne)
  • Prenotazione visuale dei posti con colori per stato
  • Notifiche email automatiche per conferme prenotazioni
  • Ricerca avanzata prenotazioni per amministratori
  • Database cloud su Neon PostgreSQL

Tipologie di utente

RuoloDescrizionePermessi
Utente Partecipante agli eventi Visualizza calendario, prenota posti, riceve email di conferma
Admin Organizzatore / Gestore Crea sale, crea eventi, gestisce prenotazioni, ricerca partecipanti, vede nomi sui posti

👤 2. Guida Utente

Questa sezione spiega come utilizzare EventBooking come partecipante agli eventi.

Utente 2.1 Registrazione e Login

Registrazione

1

Accedi alla pagina di registrazione

Clicca su Registrati nella barra di navigazione.

2

Compila il modulo

Inserisci:

  • Nome e Cognome: il tuo nome completo
  • Email: indirizzo valido (serve per le conferme)
  • Cellulare: numero di telefono (opzionale ma consigliato)
  • Password: almeno 8 caratteri
3

Conferma

Clicca Registrati. Verrai reindirizzato alla pagina di login.

Nota: Il primo account admin deve essere creato manualmente nel database o tramite procedura dedicata.

Login

1

Inserisci le credenziali

Usa l'email e la password scelti in fase di registrazione.

2

Accesso al calendario

Dopo il login vedrai il calendario degli eventi.

Utente 2.2 Il Calendario Eventi

Il calendario e' la schermata principale dell'applicazione. Mostra gli eventi programmati mese per mese.

Navigazione

Utente 2.3 Prenotare un Posto

La prenotazione avviene tramite una mappa visiva della sala, dove ogni posto e' rappresentato da un riquadro colorato.

Legenda colori

ColoreStatoSignificato
Bianco Libero Posto disponibile, cliccabile
Arancione Riservato Posto temporaneamente bloccato dall'amministratore
Azzurro Prenotato Posto gia' occupato
Verde Acqua In selezione Posti che stai selezionando (clicca di nuovo per deselezionare)
Viola Abbonato Posto riservato agli abbonati (gestito solo dall'amministratore)
Nuovo: Il colore In selezione e' stato aggiornato a Verde Acqua per una migliore distinzione visiva rispetto all'arancione dei posti riservati.

Procedura di prenotazione

1

Seleziona l'evento

Dal calendario, clicca su una data con evento e poi su Prenota.

2

Scegli i posti

Clicca sui posti bianchi (liberi) per selezionarli. Diventeranno verdi acqua.

Puoi selezionare uno o piu' posti contemporaneamente.

Per deselezionare un posto, cliccalo di nuovo.

3

Conferma

Clicca Conferma Prenotazione. I posti diventeranno azzurri (prenotati).

4

Email di conferma

Riceverai un'email all'indirizzo registrato con i dettagli della prenotazione:

  • Nome dell'evento
  • Data e ora
  • Sala
  • Posti prenotati (es. A1, A2, B3)
Attenzione: Una volta confermata, la prenotazione non puo' essere annullata direttamente dall'utente. Contatta un amministratore per modifiche.
[Screenshot: Layout sala con posti colorati, fila A, B, C... e colonne numerate]

Utente 2.4 Visualizzare Eventi

Doppio click su data con evento

  • Nome dell'evento

  • Nome della sala
  • Pulsante per prenotare

🔧 3. Guida Amministratore

Config 3.1 Installazione e Configurazione

Requisiti di sistema

  • Python 3.9 o superiore
  • Connessione Internet (per database Neon e email)

Installazione passo-passo

1

Scarica il progetto

Estrai l'archivio in una cartella sul tuo PC.

2

Crea l'ambiente virtuale

cd event_booking_app
python -m venv venv

Su Windows:

venv\Scriptsctivate

Su Linux/Mac:

source venv/bin/activate
3

Installa le dipendenze

pip install -r requirements.txt
4

Crea il file .env

Nella stessa cartella di app.py, crea un file chiamato .env:

SECRET_KEY=una-chiave-sicura-lunga-e-casuale-123456789
DATABASE_URL=postgresql://user:password@host.neon.tech/db?sslmode=require
MAIL_SERVER=smtp.gmail.com
MAIL_PORT=587
MAIL_USE_TLS=true
MAIL_USERNAME=tua-email@gmail.com
MAIL_PASSWORD=la-tua-app-password
MAIL_DEFAULT_SENDER=tua-email@gmail.com
Importante: Non condividere mai il file .env o committarlo su GitHub! Aggiungilo al .gitignore.

Config 3.2 Configurazione Database Neon

Creare un database su Neon

1

Registrati su Neon

Vai su neon.tech e crea un account gratuito.

2

Crea un nuovo progetto

Clicca New Project, scegli il nome e la regione piu' vicina a te.

3

Ottieni la stringa di connessione

Nella dashboard di Neon, vai su Connection Details e copia la stringa:

postgresql://user:password@ep-xxx.neon.tech/neondb?sslmode=require

Incollala nel file .env come valore di DATABASE_URL.

4

Crea le tabelle

Avvia l'applicazione e visita (una sola volta):

http://localhost:5000/init-db
Attenzione: Dopo la prima inizializzazione, rimuovi o proteggi la route /init-db in produzione.

Admin 3.5 Gestione Sale

Le sale sono gli spazi fisici dove si tengono gli eventi.

Nota: Al momento la creazione delle sale richiede l'inserimento diretto nel database o tramite script.

Inserire una sala tramite Python

from app import app
from models import db, Sala

with app.app_context():
    sala = Sala(
        nome="Sala Conferenze A",
        descrizione="Sala principale con proiettore e audio",
        indirizzo="Via Roma 123, Milano",
        posti_max=120,
        email_admin="admin1@example.com, admin2@example.com"
    )
    db.session.add(sala)
    db.session.commit()
    print(f"Sala '{sala.nome}' creata con ID {sala.id}")

Campi della tabella Sala

CampoTipoDescrizione
nomeTestoNome identificativo della sala
descrizioneTestoDettagli, dotazioni, note
indirizzoTestoIndirizzo fisico della sala
posti_maxInteroCapacita' massima totale
email_adminTestoEmail separate da virgola per notifiche

Admin 3.6 Creazione Eventi

Gli amministratori possono creare eventi selezionando una data futura dal calendario.

Procedura

1

Seleziona la data

Dal calendario, clicca su un giorno futuro.

2

Clicca "Crea Evento"

Verrai reindirizzato al modulo di creazione con la data precompilata.

3

Compila il modulo

CampoDescrizione
SalaSeleziona la sala dall'elenco
Numero FileQuante file di posti (max 26, corrispondono alle lettere A-Z)
Numero ColonneQuante colonne per fila
Nome EventoTitolo dell'evento
DescrizioneDettagli opzionali
Data EventoPrecompilata, modificabile solo in date future
Ora InizioOrario di inizio (formato HH:MM)
DurataDurata in minuti (default 120)
4

Verifica la capacita'

L'applicazione calcola automaticamente i posti totali (file x colonne).

Se i posti creati superano la capacita' della sala, l'evento non verra' creato.
5

Conferma creazione

Clicca Crea Evento. I posti verranno generati automaticamente con file A, B, C... e colonne numerate.

Admin 3.9 Posti Abbonato

I posti Abbonato sono una tipologia speciale di posto riservato, gestibile esclusivamente dagli amministratori. Funzionano in modo simile ai posti "Riservati" ma con un colore distintivo (viola) per identificarli facilmente nella mappa della sala.

Caratteristiche

  • Visibilità: visibili a tutti gli utenti con colore viola
  • Prenotabilità: gli utenti normali non possono prenotarli
  • Gestione: solo gli admin possono creare o cancellare posti abbonato
  • Email: le notifiche di cancellazione funzionano come per le altre prenotazioni

Procedura per abbonare posti

1

Seleziona l'evento

Dal calendario, accedi alla pagina di prenotazione dell'evento desiderato.

2

Seleziona i posti liberi

Clicca sui posti bianchi (liberi) che vuoi riservare agli abbonati.

3

Clicca "Abbona Posti"

Il pulsante viola 🎫 Abbona Posti appare solo agli admin. Cliccalo per confermare.

4

Inserisci il nome (opzionale)

Puoi personalizzare il nome dell'abbonamento, oppure lasciare il default "Abbonato".

Cancellare posti abbonato

Gli admin possono cancellare i posti abbonato cliccandoli (apparirà l'animazione rossa di selezione per cancellazione) e poi premendo 🗑️ Cancella Selezionati.

Nota: I posti abbonato condividono la logica dei posti riservati ma usano lo stato abbonato nel database e il colore #8b5cf6 (viola) nell'interfaccia.

❓ 5. FAQ e Troubleshooting

Problemi comuni

🔴 L'applicazione non si avvia

Verifica:

  • L'ambiente virtuale e' attivo?
  • Hai installato tutte le dipendenze?
  • La porta 5000 e' libera?

🔴 Errore di connessione al database

Verifica:

  • La stringa DATABASE_URL nel file .env e' corretta?
  • Hai incluso ?sslmode=require?
  • Hai eseguito /init-db almeno una volta?

🔴 Le email non arrivano

Verifica:

  • Stai usando una App Password Gmail?
  • Le credenziali nel .env sono corrette?
  • Controlla la cartella Spam.

🟡 Non vedo il pulsante "Crea Evento"

Il pulsante appare solo agli utenti con tipo admin.

🟡 Non posso prenotare posti

  • L'evento e' in una data passata
  • I posti sono gia' tutti prenotati
  • Non hai effettuato il login

🟢 Come faccio il backup del database?

pg_dump "DATABASE_URL" > backup.sql

🟢 Posso usare un altro database?

Si', modifica la stringa DATABASE_URL. Compatibile con PostgreSQL, MySQL, SQLite.
